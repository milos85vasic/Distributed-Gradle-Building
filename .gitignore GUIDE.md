# Global Git Ignore Configuration

## Overview

This repository contains multiple `.gitignore` files tailored to different parts of the distributed build system:

1. **`.gitignore`** (root) - Main project-wide ignore rules
2. **`go/.gitignore`** - Go-specific ignores
3. **`scripts/.gitignore`** - Bash scripts and tools ignores
4. **`docs/.gitignore`** - Documentation build ignores
5. **`.git/info/exclude`** - Local machine specific excludes

## Usage

### Repository Level (Already Applied)
The main `.gitignore` file is applied automatically to the entire repository and includes:
- Build artifacts (Java, Go, Gradle)
- IDE files (IntelliJ, VS Code, Eclipse)
- OS files (macOS, Windows, Linux)
- Log files and temporary data
- Sensitive configuration and credentials

### Personal Local Excludes (Optional)
Add machine-specific ignores to `.git/info/exclude` (already configured):
```bash
# Already contains comprehensive local excludes
# Add your personal patterns as needed
```

### Global Git Ignore (Optional)
Set up global ignores for all your projects:
```bash
# Create global gitignore
git config --global core.excludesfile ~/.gitignore_global

# Copy root patterns
cp .gitignore ~/.gitignore_global
```

## Ignore Pattern Categories

### Development Artifacts
- **Build outputs**: `build/`, `dist/`, `bin/`, `target/`
- **Compiled files**: `*.class`, `*.jar`, `*.exe`, `*.dll`, `*.so`
- **Gradle**: `.gradle/`, `gradle.properties.local`
- **Go**: `vendor/`, `go.work`, `*.test`, `*.out`

### Development Environment
- **IDE files**: `.idea/`, `.vscode/`, `*.iml`, `*.ipr`
- **Editor files**: `*.swp`, `*.swo`, `*~`
- **OS files**: `.DS_Store`, `Thumbs.db`, `._*`

### Runtime Data
- **Logs**: `*.log`, `logs/`
- **Temp files**: `tmp/`, `temp/`, `*.tmp`
- **Cache**: `cache/`, `gradle-cache/`, `data/`
- **Process files**: `*.pid`, `*.lock`

### Sensitive Information
- **Credentials**: `*.pem`, `*.key`, `*.p12`, `*.jks`
- **Configuration**: `*.env`, `config.local.properties`, `secrets/`
- **SSH**: `id_rsa*`, `known_hosts`

### Testing & Performance
- **Test output**: `test-results/`, `coverage/`
- **Profiling**: `*.prof`, `*.pprof`, `perf-results/`
- **Benchmarks**: `benchmark/`

### Documentation
- **Build output**: `docs/_build/`, `docs/_site/`, `docs/build/`
- **Generators**: Node modules, Python cache, etc.

## Security Considerations

The `.gitignore` files specifically target:
- **Private keys** and certificates
- **Database files** with sensitive data
- **Configuration files** with passwords/tokens
- **SSH keys** and known hosts
- **Environment files** with secrets

## Best Practices

1. **Never commit sensitive data** - Rely on `.gitignore` patterns
2. **Use template files** instead of real configs (e.g., `config.properties.template`)
3. **Test your ignores**: Use `git check-ignore` to verify patterns
4. **Commit `.gitignore` early** - Before adding sensitive files
5. **Regular updates** - Add new ignore patterns as needed

## Common Issues & Solutions

### Already Committed Files
If sensitive files were committed before `.gitignore`:
```bash
# Remove from git but keep locally
git rm --cached sensitive-file.txt

# Remove entire directory from git
git rm -r --cached sensitive-directory/

# Commit the removal
git commit -m "Remove sensitive files, add to gitignore"
```

### Check If Files Are Ignored
```bash
# Check specific file
git check-ignore path/to/file.txt

# List all ignored files
git status --ignored
```

### Temporary Override
```bash
# Force add ignored file (use carefully)
git add -f path/to/ignored-file.txt

# Commit with force
git commit -m "Add ignored file"
```

## Migration from Existing Projects

When importing existing projects:
1. Copy appropriate `.gitignore` patterns
2. Run `git status` to identify untracked files
3. Review and add any new patterns as needed
4. Remove committed artifacts that should be ignored
